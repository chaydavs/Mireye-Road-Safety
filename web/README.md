# Subgrade web — Vercel (Next.js) frontend

A static-first map UI (no Python server at runtime): the map, why-card, RSL, and live status all run
client-side from pre-generated JSON. The **only** dynamic piece is the county copilot, a single Vercel
serverless function (`app/api/copilot/route.ts`) that calls Anthropic.

## Data

The app reads three files from `public/data/`, generated by the Python pipeline:

```
# from the repo root, after the scoring pipeline + paving + RSL + ablation have run:
.venv/bin/python src/ablation.py     # traffic-only vs +Mireye priority ranking
.venv/bin/python src/export_web.py
# writes web/public/data/{segments.geojson, scores.json, live.json, summary.json, ablation.json}
```

- `segments.geojson` — geometry + relative-percentile color + top-5 cited drivers + RSL per segment
- `scores.json` — the same records without geometry (read by the copilot function server-side)
- `live.json` — live-stress status counts + watched segment ids + per-segment triggers

Regenerate `live.json` (and the snapshot) any time by re-running `src/live.py` then `src/export_web.py`.

## Local dev

```
cd web
npm install
echo "ANTHROPIC_API_KEY=sk-ant-..." > .env.local   # only needed for the copilot; gitignored
npm run dev        # http://localhost:3000
```

## Deploy to Vercel

```
cd web
vercel            # first run: log in + link the project
vercel --prod
```

Then set **`ANTHROPIC_API_KEY`** in the Vercel project (Settings → Environment Variables). Without it
the map/why-card/RSL/live all still work; only the copilot returns a "not set" message.

Notes:
- The basemap is CARTO Positron raster tiles (no API key).
- `public/data/*` is committed so a fresh deploy has data; re-run `src/export_web.py` to refresh it.
- The copilot function reads `scores.json` from its own origin and caches it across warm invocations.
